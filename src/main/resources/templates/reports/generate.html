🧵 How Report Generation Works

  • CO1 — @Async: The generation runs in a separate thread pool (ReportAggregator-N), not blocking the web request.
  • CO1 — ThreadPoolExecutor: Each industry's data is aggregated in parallel (4 core threads, 8 max).
  • CO3 — @Transactional: Hibernate aggregates HQL queries across waste_entries table.
  • CO4 — Socket Alert: On completion, a real-time alert is broadcast to all connected regulators via AlertServer (port 9090).
